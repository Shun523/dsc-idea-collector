# のがにゃんを唸らせろ

**岡山大学DS部の部内DX推進を目的とした、アイデア投稿アプリです。**

部員が日常業務の改善案や新しい取り組みのアイデアを気軽に投稿できる場を作ることが目的です。「アイデアを書いて投稿する」というハードルを下げるため、顧問「のがにゃん」に手紙を届けるという演出と、返ってきたコメントを集める **コレクション要素** を組み込んでいます。

部室常設のiPadから投稿することを想定しています。

---

## 仕組み

1. **投稿タブ** でアイデアやメッセージを書いて送信
2. 便箋が封筒に入り、DSCスタンプが押されてのがにゃんに飛んでいくアニメーションが流れる
3. AIがメッセージのタイプを判定し、**のがにゃんが過去にSlackで実際に発言したコメント** が吹き出しで返ってくる
4. **過去のアイデアタブ** で過去に投稿されたアイデアを封筒のサムネイルから見返せる
5. **コレクションタブ** で、これまで引き出したのがにゃんの発言を確認できる

> 「アイデアを真面目に書く」だけでなく、「どんなコメントを引き出せるか試してみる」というモチベーションも生まれることで、投稿の心理的ハードルを下げる狙いがあります。

## 画面

| タブ | 内容 |
|---|---|
| 投稿 | のがにゃん（中央）と入力フォーム（下）。送信すると封筒アニメーション → 到着シーン |
| 過去のアイデア | 過去の投稿を封筒のサムネイルでグリッド表示。タップで本文と返信コメントを表示 |
| コレクション | 顧問のSlack発言を集めたカード一覧。引き当てた発言だけが見える（未取得はマスク） |

## 技術スタック

| 領域 | 採用技術 |
|---|---|
| フロントエンド | React 19 + TypeScript（Vite） |

| バックエンド | Node.js + Express + TypeScript |
| AI | Google Gemini API（`@google/generative-ai`） |
| データ保存 | JSONファイル（`backend/data/`） |

詳細な仕様は [`spec.md`](./spec.md) を参照。

## セットアップ

### 1. 必要なもの

- Node.js（20以上推奨）
- Google Gemini API キー（[Google AI Studio](https://aistudio.google.com/apikey) で発行）

### 2. 依存パッケージのインストール

```bash
# バックエンド
cd backend
npm install

# フロントエンド
cd ../frontend
npm install
```

### 3. 環境変数の設定

`backend/.env` を作成：

```env
GEMINI_API_KEY=ここにAPIキーを貼る
PORT=3001
```

> ⚠️ `.env` は絶対にコミットしないこと（`.gitignore` 済み）。

### 4. 起動

ターミナルを2つ開いて、それぞれで：

```bash
# ターミナル1：バックエンド
cd backend
npm run dev

# ターミナル2：フロントエンド
cd frontend
npm run dev
```

ブラウザで http://localhost:5173 を開く。

### 5. iPad から接続する

PCとiPadを同じWi-Fiに繋ぎ、PCのローカルIPアドレスを使ってアクセス：

```
http://<PCのIPアドレス>:5173
```

## 開発のポイント

- **テストモード**：ヘッダー右のチェックボックスをオンにすると、Gemini API を呼ばずにモックデータでアニメーションを試せる（開発・動作確認用）
- アニメーションは Motion の `phase` 状態機械で多段階制御している（`form` → `inserting` → `closing` → `stamping` → `flying` → `arrived`）
