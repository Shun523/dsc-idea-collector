## 0. この仕様書の使い方（重要）

このアプリは **バイブコーディングなし** で、Claude Codeに質問しながら手で書いて学ぶことを方針とします。

### 学習の進め方

- Claude Codeに「全部書いて」と頼まない
- 「この機能を実装するにはどのファイルをどう変えればいい？」のように **1ステップずつ** 聞く
- コードを手で打ち込み、動かして、エラーが出たら自分でエラーを読んでから質問する
- 「なぜこう書くの？」を必ず聞く習慣をつける
- 詰まったらAIに聞けばOK。ただし「なぜそうなるのか」まで理解してから次に進む

### Claude Codeへの質問テンプレート

```
【状況】○○を実装しようとしている
【やったこと】△△というコードを書いた
【エラー/問題】□□というエラーが出た（またはうまく動かない）
【聞きたいこと】どこが間違っていますか？
```

## 1. プロジェクト概要

岡山大学DS（データサイエンス）部の部内DX推進を目的に、部員がアイデアを気軽に投稿できるインタラクティブなWebアプリを開発する。アイデアを投稿すると風船アニメーションとともに画面上に表示され、AIがアイデアのすばらしさを評価してランク付けし、ランクに応じた顧問の一言コメントが表示される。

## 2. フロントエンドとバックエンドの関係

### 構成図

```
┌──────────────────────────┐          ┌──────────────────────────┐
│      フロントエンド          │          │       バックエンド           │
│   React（Vite）            │          │   Node.js + Express        │
│   localhost:5173           │  HTTP通信 │   localhost:3001           │
│                            │ ────────▶│                            │
│ ・画面の表示                │          │ ・Claude APIの呼び出し      │
│ ・ボタン・フォーム           │ ◀──────── │ ・APIキーの管理             │
│ ・風船アニメーション         │  JSONで返す│ ・データの保存・読み込み     │
│                            │          │                            │
│ ブラウザ内で動く             │          │ ターミナルで動く             │
│ （誰でもソースを見られる）    │          │ （外から中身は見えない）      │
└──────────────────────────┘          └──────────────────────────┘
```

### 開発時のイメージ

開発中はターミナルを**2つ**開いて、それぞれを起動する。

```bash
# ターミナル1：フロントエンド
cd frontend
npm run dev   → http://localhost:5173 で画面が見られる

# ターミナル2：バックエンド
cd backend
npm run dev   → http://localhost:3001 でAPIが動く
```

iPadのSafariで `http://[PCのIPアドレス]:5173` を開けばアプリが使える。

### 通信の流れ（例：アイデア送信時）

```
① iPad（Safari）でフォームに入力して「送信」ボタンを押す

② フロントエンド（React）が
   fetch("http://localhost:3001/api/submit", { method: "POST", body: アイデアのテキスト })
   でバックエンドにリクエストを送る

③ バックエンド（Express）が受け取り、Claude APIにアイデアを送って評価結果をもらう

④ バックエンドがランク・コメントをJSONでフロントに返す

⑤ フロントエンドが受け取り、風船アニメーションを表示する
```

## 3. 技術スタック

| 項目 | 技術 | なぜこれ？ |

| フロントエンド | React + TypeScript（Vite） | 現場で最もよく使われる構成。学習資料が豊富 |

| 実行環境 | ローカルPC（iPad と同じWi-Fi） | iPad から PC のサーバーにアクセスする |

## 5. 画面構成

### 5.1 メイン画面

```
┌──────────────────────────────────────┐
│  🎈  DS部 アイデアボックス  🎈          │
│                                      │
│  （過去の風船がふわふわ浮いている）       │
│  🎈        🎈                        │
│      🎈          🎈                  │
│                                      │
│  ┌──────────────────────────────┐    │
│  │ あなたのアイデアを書いてね...   │    │
│  │                              │    │
│  └──────────────────────────────┘    │
│            [送信する 🚀]              │
└──────────────────────────────────────┘
```

- 背景に過去の投稿アイデアが風船として浮遊（最大20個）
- テキストエリア（最大300文字）
- 送信ボタン

### 5.2 送信後のアニメーション

1. 送信ボタン押下 → 「AIが評価中...」ローディング表示
2. 評価完了 → 画面下部から風船が上昇
3. 中央で一時停止 → 吹き出しで顧問コメントが表示（タイプライター風）
4. 数秒後 → 風船が背景の浮遊グループに合流

### 5.3 管理画面（`/admin`）

- 顧問コメントの一覧・追加・編集・削除
- 各コメントの出現確率（weight）設定
- 投稿一覧の確認
- パスワード認証あり

## 6. 機能仕様

### 6.1 アイデア投稿

- テキストエリアに入力（ひらがな・漢字・英語対応）
- バリデーション：空文字は送信不可 / 最大300文字
- 送信中はボタンをグレーアウト + ローディング表示

### 6.2 AI評価（Claude API）

バックエンドの `/api/submit` ルートから Claude API を呼び出す。

#### Claude に渡すプロンプト

```
あなたは岡山大学DS部（データサイエンス部）のアイデア審査AIです。
部員から投稿されたアイデアを評価し、ランクを付けてください。

【投稿されたアイデア】
{ユーザーの入力テキスト}

【評価基準】
- DS部の活動として取り組める実現可能性があるか
- 新しい視点・発想がある独創性があるか
- 部や大学・社会への貢献が期待できるか

【ランク定義】
S: 革新的で即実行したいレベルの超優秀アイデア
A: 実用性が高く、ぜひ取り組みたい優秀アイデア
B: 良いアイデア。工夫次第で発展の可能性あり
C: まずまずのアイデア。もう少し具体化を
D: 基本的なアイデア。ブラッシュアップが必要

必ず以下のJSON形式のみで回答してください（説明文は不要）:
{
  "rank": "S" | "A" | "B" | "C" | "D",
  "score": 0〜100の整数,
  "reason": "評価理由（50文字以内の日本語）"
}
```

### 6.3 ランクに応じたビジュアル

| ランク | 風船色 | 風船サイズ | 演出 |
|--------|--------|-----------|------|
| S | 金色グラデーション | 特大 | 紙吹雪 + キラキラ |
| A | 赤 | 大 | 星エフェクト |
| B | 青 | 中 | 通常浮遊 |
| C | 緑 | 中（やや小） | 通常浮遊 |
| D | グレー | 小 | ゆっくり浮遊 |

風船をタップすると、そのアイデアのテキストと顧問コメントがポップアップ表示される。

### 6.4 顧問コメントの抽選

同一ランク内で `weight` による重み付き抽選を行う。

#### データ形式（`backend/data/comments.json`）

```json
{
  "comments": [
    { "id": "s001", "rank": "S", "text": "これは採用だ！来週の部会で発表してくれ！", "weight": 30 },
    { "id": "s002", "rank": "S", "text": "君はDS部の未来だ。", "weight": 20 },
    { "id": "a001", "rank": "A", "text": "いいね！もう少し肉付けして提案書にまとめてみて。", "weight": 40 }
  ]
}
```

`weight` の合計に対する比率で出現確率が決まる。例えば同じランクで weight が 30 と 20 なら、それぞれ 60% と 40% の確率で選ばれる。

## 7. APIエンドポイント設計（バックエンド）

| メソッド | パス | 説明 |
|---------|------|------|
| POST | `/api/submit` | アイデア受け取り・AI評価・コメント抽選・保存 |
| GET | `/api/posts` | 背景風船用の最近の投稿一覧 |
| GET | `/api/admin/comments` | 顧問コメント一覧取得 |
| POST | `/api/admin/comments` | コメント追加 |
| PUT | `/api/admin/comments/:id` | コメント更新 |
| DELETE | `/api/admin/comments/:id` | コメント削除 |
| GET | `/api/admin/posts` | 全投稿一覧（管理用） |

### POST `/api/submit` の例

リクエスト（フロントから送る）:
```json
{ "text": "部員のスケジュール管理にAIを使いたい" }
```

レスポンス（バックから返ってくる）:
```json
{
  "id": "post_042",
  "text": "部員のスケジュール管理にAIを使いたい",
  "rank": "A",
  "score": 78,
  "reason": "DS部の実務に即した実用的な提案",
  "comment": "いいね！もう少し肉付けして提案書にまとめてみて。",
  "createdAt": "2026-04-24T15:30:00+09:00"
}
```

## 9. CORS設定について

フロント（port:5173）とバック（port:3001）はポートが違うため、そのままだとブラウザがセキュリティエラーを出す（**CORSエラー**）。バックエンド側で `cors` パッケージを使って許可する必要がある。

```typescript
// backend/src/index.ts
import cors from "cors";
app.use(cors({ origin: "http://localhost:5173" }));
```

> 💡 CORSエラーが出たときは「なぜこのエラーが出るのか？」をClaude Codeに聞いてみよう。フロントとバックを分ける構成ならではの重要な学びポイント。

## 10. iPad対応要件

- タッチ操作しやすいボタンサイズ（最小44×44px）
- フォントサイズ最小16px（Safariの自動ズーム防止）
- 縦向き（Portrait）メインで設計
- iPadからPCのサーバーにアクセスするには同じWi-Fiに接続し、PCのローカルIPアドレスを使う
  ```
  例: http://192.168.1.5:5173
  ```

## 12. 実装ステップ（学習ロードマップ）

| ステップ | 作業内容 | 主な学び |
|---------|---------|---------|
| 1 | バックエンド：Expressで "Hello" を返すだけのサーバー | Node.js, Express の基礎 |
| 2 | バックエンド：POST でテキストを受け取ってログ出力 | リクエストの受け取り方 |
| 3 | バックエンド：Claude APIを呼び出してランクを返す | 外部API連携、環境変数 |
| 4 | バックエンド：JSONファイルへの保存・読み込み | ファイルI/O |
| 5 | フロントエンド：フォームとボタンの表示 | React, useState |
| 6 | フロントエンド：バックエンドにfetchで送信 | fetch, async/await, CORS |
| 7 | フロントエンド：風船アニメーションの実装 | CSS animation, 条件レンダリング |
| 8 | フロントエンド：背景の浮遊風船 | 状態管理, useEffect |
| 9 | 管理画面の作成 | ルーティング, 認証 |

> バックエンドから先に作るのがおすすめ。APIが固まってからフロントを作ると、何を受け取って何を返すかが明確になる。
